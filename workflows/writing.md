# Writing Guidance

Write with concise, direct technical prose and no filler. Do not use emojis in
commits, code, skill files, task files, plans, or convention documents.

Prefer concise prose over lists. Use bullets only when they make concrete
requirements, constraints, findings, steps, or distinct cases easier to scan.

Focus on what and why before how. Include implementation detail only when it is
necessary to describe a requirement, constraint, interface contract, risk, or
verification point.
