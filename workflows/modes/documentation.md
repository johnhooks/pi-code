# Documentation Mode

Use this mode when the task primarily changes documentation, examples, README
content, or inline explanatory comments.

Documentation should be useful, current, and grounded in project evidence.

## Expectations

Read the relevant source or existing docs before writing. Prefer concise
explanations over exhaustive background. Keep setup and workflow instructions
direct and current.

Distinguish confirmed behavior from planned or speculative behavior. Update
examples when behavior, commands, or file paths change.

## Done Means

The documentation is accurate to the current implementation. Instructions are
concise enough to follow. Links, commands, and file paths are checked when
practical. Speculative or future-facing language is clearly marked or omitted.
