# Code Cleanup Mode

Perform one scoped cleanup pass after implementation review.

Use the review findings to improve the changed code while staying inside the
approved task. Remove unnecessary code, tighten names, simplify control flow,
update docs or comments when needed, and keep the final diff readable.

Complete the `test` responsibilities before the final report: choose relevant
verification from project evidence, run it when safe, and interpret the result.
If verification cannot be run, explain why and identify what should be run.

Report the final summary, verification, material deviations from the approved
plan, and any remaining follow-up.
